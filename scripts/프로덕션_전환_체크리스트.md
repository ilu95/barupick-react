## Step 1: 프로덕션 DB 스키마 패치

### 1-1. Supabase Dashboard 접속
- URL: https://supabase.com/dashboard/project/kwcogjzwpnvqwmifizce
- SQL Editor 열기

### 1-2. 스키마 SQL 실행
- `scripts/01_schema.sql` 파일의 SQL을 **전체 복사하여 실행**
- 이미 테이블이 있으면 `CREATE TABLE IF NOT EXISTS`로 안전하게 스킵됨
- **핵심**: `handle_new_user` 트리거가 EXCEPTION WHEN OTHERS 없이 새로 생성됨

### 1-3. 실행 후 확인
SQL Editor에서 아래 쿼리로 확인:
```sql
-- 테이블 존재 확인
SELECT table_name FROM information_schema.tables 
WHERE table_schema = 'public' 
ORDER BY table_name;

-- 예상 결과: blocks, bookmarks, comments, event_submissions, events, 
--           feedbacks, follows, likes, notifications, posts, profiles, 
--           reports, user_data

-- 트리거 확인
SELECT trigger_name, event_manipulation, action_statement 
FROM information_schema.triggers 
WHERE trigger_schema = 'public' OR event_object_table = 'users';

-- RLS 확인
SELECT tablename, policyname FROM pg_policies WHERE schemaname = 'public';
```

### 1-4. Storage 버킷 확인
Dashboard → Storage에서 아래 2개 버킷이 있는지 확인:
- `avatars` (Public)
- `community` (Public)
- 없으면 "New Bucket" → 이름 입력 → Public 체크

## Step 2: 소셜 로그인 설정

### 2-1. 카카오 로그인
1. [Kakao Developers](https://developers.kakao.com) → 내 애플리케이션
2. 앱 설정 → 플랫폼 → Web → 사이트 도메인:
   - `https://barupick-react.vercel.app` (또는 최종 도메인)
3. 카카오 로그인 → Redirect URI 추가:
   - `https://kwcogjzwpnvqwmifizce.supabase.co/auth/v1/callback`
4. Supabase Dashboard → Authentication → Providers → Kakao:
   - Client ID: 카카오 앱 REST API 키
   - Client Secret: 카카오 앱 Secret Key
   - Redirect URL 확인: `https://kwcogjzwpnvqwmifizce.supabase.co/auth/v1/callback`

### 2-2. 구글 로그인
1. [Google Cloud Console](https://console.cloud.google.com) → OAuth 동의 화면 + 사용자 인증 정보
2. 승인된 리디렉션 URI 추가:
   - `https://kwcogjzwpnvqwmifizce.supabase.co/auth/v1/callback`
3. Supabase Dashboard → Authentication → Providers → Google:
   - Client ID / Client Secret 입력

### 2-3. 이메일 로그인
Supabase Dashboard → Authentication → Providers → Email:
- Enable Email provider ✓
- Confirm email: ON (선택) 또는 OFF (가입 즉시 사용)

| Key | Value | 환경 |
|-----|-------|------|
| `VITE_SUPABASE_URL` | `https://kwcogjzwpnvqwmifizce.supabase.co` | Production |

## Step 4: 도메인 전환

### 옵션 A: barupick.vercel.app 이전 (간단)
1. Vercel Dashboard → 기존 HTML 프로젝트에서 `barupick.vercel.app` 도메인 제거
2. React 프로젝트에 `barupick.vercel.app` 도메인 추가
3. ⚠️ **같은 도메인이면 localStorage 자동 유지** — 마이그레이션 스크립트 자동 실행

### 옵션 B: 커스텀 도메인 (barupick.co.kr)
1. Vercel Dashboard → React 프로젝트 → Settings → Domains → Add `barupick.co.kr`
2. DNS 설정 (A Record → 76.76.21.21 또는 CNAME → cname.vercel-dns.com)
3. ⚠️ **다른 도메인이면 localStorage 데이터 없음** — 기존 사용자 안내 필요

### 옵션 C: 점진적 전환 (가장 안전)
1. React 버전을 `app.barupick.vercel.app` 등 서브도메인에서 먼저 운영
2. 충분히 검증 후 메인 도메인으로 교체

## Step 5: 기존 사용자 데이터 처리

### 5-1. localStorage 마이그레이션 (자동)
`main.tsx`에 마이그레이션 코드가 포함되어 있어 **첫 로드 시 자동 실행**:
- 옷장 아이템 `colorKey` → `color` 통합
- OOTD 기록 `id` 없는 레코드에 id 생성
- `sp_migrated_v2` 플래그로 1회만 실행

### 5-2. Supabase Auth 세션
- **같은 Supabase 프로젝트면**: 기존 사용자 로그인 세션 유지
- **다른 Supabase 프로젝트면**: 사용자 재로그인 필요
  - 프로덕션 DB(kwcogjzwpnvqwmifizce)에는 기존 사용자 데이터가 이미 있을 수 있음
  - 개발 DB(ywqaxxcvzhwhascbkyhp) 사용자 데이터는 별도

### 5-3. 공지 (선택)
기존 HTML 버전 사용자에게 전환 안내:
- 인스타 스토리 or 바루사 매거진에 공지
- "바루픽이 새롭게 업데이트되었습니다" 메시지
